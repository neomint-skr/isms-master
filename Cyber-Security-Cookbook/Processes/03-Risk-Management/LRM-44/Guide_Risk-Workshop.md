# Risk Workshop Guide (LRM-44)

## Purpose

This guide enables human-executable, deterministic risk workshops using LRM-44 (4×4). It standardizes preparation, facilitation and documentation.

## Preparation (before the workshop)

1. Confirm scope unit definition and boundaries.
2. Collect basic context:
   - external exposure
   - user groups
   - critical dependencies
   - recent major changes/incidents
3. Prepare scenario candidates from:
   - `ISMS-Handbook/Registers/05-Risk-Library/Library_Scenarios.md`

## Workshop agenda (60–90 minutes)

1. Confirm scope (5 min)
2. Select material scenarios (10–20 min)
3. Score each scenario (Impact + Likelihood) using the binding rubrics (25–40 min)
4. Decide (accept/treat/transfer/avoid) using acceptance criteria (10–15 min)
5. For treated risks: select controls and define 1–3 actions with due dates (10–15 min)
6. Confirm approvals and follow-ups (5 min)

## Documentation (mandatory)

- Scope unit assessment: use `Cyber-Security-Cookbook/Templates/21-Risk-Assessment.md`
- Risk Log update: use `Cyber-Security-Cookbook/Templates/22-Risk-Log.md`
- Workshop minutes: use `Cyber-Security-Cookbook/Templates/24-Risk-Workshop-Minutes.md`

## Facilitation rules (determinism)

- Use the rubrics as written; do not invent local scales.
- If uncertain, document the assumption and choose the more conservative rating.
- Keep scenarios one sentence; avoid multi-part scenarios in one entry.
